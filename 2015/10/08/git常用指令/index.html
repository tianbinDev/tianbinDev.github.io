git常用指令

发表于 2015-10-08   |   分类于 Git

安装

sudo apt-get install git

用户信息配置

git config --global user.name "Your Name"
git config --global user.email "email@example.com"

常用指令使用说明

命令 功能
git init 将当前目录作为git仓库
git add 将文件添加到仓库
git commit 将文件提交到仓库
git status 查看当前仓库状态
git diff 查看修改内容
git log 查看提交日志
git reset –hard HEAD^ 回退到上一个版本
git reset –hard commit_id 回退到commit_id之前版本
git reflog 查看执行过的命令
git checkout –file 丢弃文件在工作区的修改
git reset HEAD file 丢弃文件在暂存区的修改
git remote add origin git@server-name:path/repo-name.git 关联一个远程仓库
git push origin master 推送修改到远程仓库
git branch 查看分支
git branch 创建分支
git checkout 切换分支
git checkout -b 创建+切换分支
git merge 合并某分支到当前分支
git branch -d 删除分支
git stash 储存工作现场
git stash list 查看储存的现场
git stash apply/git stash pop 恢复工作现场
git remote -v 查看远程库信息
git tag 创建标签
git tag 查看所有标签
git tag -a -m “tag content” 指定标签信息
git tag -s -m “tag content” 用PGP签名标签
git push origin 推送tagname标签
git push origin –tags 推送所有未推送的标签
git tag -d 删除tagname标签
git push origin :refs/tags/ 删除远程标签
git config –global color.ui true 让git输出内容显示颜色
git config –global alias.st status 配置别名

注意事项

  • git stash 命令可以将本地修改进行暂存，该暂存区是属于仓库的，而不是属于分支。所以恢复修改时需要使用git stash apply <stashname>
  • 提交到远端的代码不要再使用git rebase进行重新合并
