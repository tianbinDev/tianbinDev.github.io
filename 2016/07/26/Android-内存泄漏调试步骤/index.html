Android 内存泄漏调试步骤

发表于 2016-07-26   |   分类于 Android

使用 Android Device Monitor 及 MAT 调试分析 Android 内存泄漏

调试步骤：

  • 安装应用到移动设备
  • 打开 Android Device Monitor (Tools –> Android –> Android Device Monitor)
  • 进入可能存在内存泄漏的页面，然后退出，重复操作几次
  • 选中 Devices 列表中的应用包名，点击 update Heap 图标，在 Heap 标签页中可以看到内存占用情况
  • 点击 Cause GC 手动回收内存
  • 点击 Dump HPROF file 导出内存分析文件

分析步骤：

  • 下载 MAT
  • 使用命令将导出的 hprof 文件转成 MAT 软件可以解析的文件 (hprof-conv file.hprof newfile.hprof)
  • 将生成的 hprof 文件导入 MAT
  • 点击 QQL 图标进入 QQL 标签页，输入select * from instance of android.app.Activity 并点击执行
  • 查询结果列表中展示出存在内存泄漏的Activity
  • 选中列表中某一项，右键选中 Path To GC Roots 后，点击 exclude all phantom/weak/soft etc. references
  • 新打开的页面中会展示出导致内存泄漏的对象
  • 分析发生内存泄漏的原因并修正

发生内存泄漏的原因：

  • Handler 使用不当，持有 Activity Context 导致的内存泄漏
  • Adapter 中 onViewDetachedFromWindow 方法没有被调用，引起的 Holder 无法被释放，从而导致 内存泄漏 (解决办法：Activity 销毁时 setAdapter(null) )
  • 自定义 View 持有 Activity Context 导致内存泄漏

遇到的Keng：

  • 使用 Android Studio 中 Android Monitors 标签页 点击 Dump Java Heap 无法导出 hprof 内存分析文件
